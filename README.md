# ADK + LiteLLM + A2A Template

A template for building agents with [Google ADK](https://github.com/google/adk-python), [LiteLLM](https://github.com/BerriAI/litellm) proxy for multi-model routing, and the [A2A (Agent-to-Agent)](https://github.com/google/A2A) protocol for standardized agent discovery and communication.

Your agent is exposed via A2A so that other agents can discover it (via its auto-generated agent card) and interact with it using the A2A protocol.

## Quick Start

### 1. Install

```bash
python3 -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt
cp .env.example .env    # fill in your keys + proxy URL
```

### 2. Run the A2A Server

```bash
uvicorn a2a_app:app --host=0.0.0.0 --port=8080
```

Verify the agent card is being served:

```bash
curl http://localhost:8080/.well-known/agent-card.json
```

You should see a JSON response with your agent's name, description, and auto-discovered skills.

### 3. Docker

```bash
docker build -t adk-agent .
docker run --env-file .env -p 8080:8080 adk-agent
```

### Local Dev with ADK Web UI

For interactive testing during development, you can still use the ADK Web UI:

```bash
adk web .    # opens ADK web UI at http://localhost:8080
```

## A2A Protocol

The [A2A protocol](https://github.com/google/A2A) is a standard for agent-to-agent communication. This template uses ADK's `to_a2a()` utility to automatically expose your agent with:

- **Agent card** at `/.well-known/agent-card.json` — auto-generated from your agent's name, description, and tools
- **A2A messaging endpoint** — other agents can send messages and receive responses via the A2A protocol

### Discovering the Agent

Any A2A-compatible consumer can discover your agent by fetching the well-known agent card:

```bash
curl http://localhost:8080/.well-known/agent-card.json
```

### Consuming from Another ADK Agent

Another ADK agent can connect to your exposed agent using `RemoteA2aAgent`:

```python
from google.adk.agents.remote_a2a_agent import RemoteA2aAgent

remote_agent = RemoteA2aAgent(
    name="remote_weather",
    description="Remote weather agent via A2A",
    agent_card_url="http://localhost:8080/.well-known/agent-card.json",
)
```

## Customizing the Agent Card

The agent card is auto-generated by default. To customize it, pass an `agent_card` parameter to `to_a2a()` in `a2a_app.py`:

**With an `AgentCard` object:**

```python
from a2a.types import AgentCard

my_card = AgentCard(
    name="weather_agent",
    url="http://localhost:8080",
    description="Weather and search assistant",
    version="1.0.0",
    capabilities={},
    skills=[],
    defaultInputModes=["text/plain"],
    defaultOutputModes=["text/plain"],
    supportsAuthenticatedExtendedCard=False,
)

app = to_a2a(root_agent, agent_card=my_card)
```

**With a JSON file:**

```python
app = to_a2a(root_agent, agent_card="/path/to/agent-card.json")
```

## How It Works

**A2A Exposure** — `a2a_app.py` imports your agent and wraps it with `to_a2a()`, which creates an ASGI application that speaks the A2A protocol. The agent card is auto-generated from the agent's metadata. Uvicorn serves this ASGI app.

**LiteLLM Proxy** runs externally (you manage it separately). This agent points at it via `LITELLM_PROXY_API_BASE` in `.env`. The proxy handles routing to OpenAI, Anthropic, Gemini, or any other provider.

**ADK Skills** load incrementally to minimize context window usage:
- **L1** (name/description) — always loaded for discovery
- **L2** (instructions) — loaded when the skill is triggered
- **L3** (references/assets) — loaded on demand when referenced

## Switching Models

Change `DEFAULT_MODEL` in `.env` to any model your LiteLLM proxy serves:

```bash
curl http://litellm.overcastlab.com:4000/models -H 'Authorization: Bearer sk-1234'
```

## Adding a Skill

1. Create `skills/my-skill/SKILL.md`:

```markdown
---
name: my-skill
description: One-line description for when to use this skill.
---

Step 1: Do something...
Step 2: Read 'references/GUIDE.md' for details.
Step 3: Return the result.
```

2. Load it in `agent.py`:

```python
my_skill = load_skill_from_dir(_SKILLS_DIR / "my-skill")
# Add to the SkillToolset skills list
```
